# Рекомендательная система контента

## Тема
Разработка и сравнительный анализ алгоритмов рекомендательной системы контента (фильмы, сериалы, книги, музыка) на основе предпочтений пользователя.

## Список требований к программе

### Функциональные требования
1. Определение возрастной группы пользователя (7 групп: дети, подростки, молодёжь, взрослые, средний возраст, старшее поколение, пожилые).
2. Учёт возрастных ограничений контента (0+, 6+, 12+, 16+, 18+).
3. Выбор предпочитаемых жанров из 23 категорий.
4. Реализация трёх алгоритмов рекомендаций:
   - Правила: возраст + жанры + целевая группа.
   - Коллаборативная фильтрация: на основе синтетических пользователей.
   - Популярность: топ элементов по рейтингу популярности.
5. Вывод рекомендаций (не более 5 позиций на категорию) для каждого алгоритма.
6. База данных содержит более 70 элементов (фильмы, сериалы, книги, музыка) с контентом.

## Описание алгоритмов

### Алгоритм 1 – на основе правил
Этот алгоритм последовательно фильтрует всю базу контента по четырём критериям:
1. Тип контента (фильм, сериал, книга или музыка) должен совпадать с запрошенным.
2. Возрастной рейтинг элемента не должен превышать максимально допустимый для пользователя (определяется по возрасту: 0–12 лет – рейтинг 6+, 13–17 – 12+, от 18 – 18+).
3. Целевая группа элемента (`child`, `teen`, `young`, `adult`, `middle`, `senior`, `elderly`) должна строго совпадать с группой пользователя.
4. Жанр элемента должен присутствовать в списке предпочтений, который пользователь выбрал в меню.
Если элемент проходит все четыре проверки, он добавляется в список рекомендаций. Алгоритм останавливается, как только набрано 5 позиций.

### Алгоритм 2 – коллаборативная фильтрация 
Работает на основе шести заранее созданных синтетических пользователей. Каждый из них имеет свой возраст, любимые жанры и оценки (от 1 до 5) для некоторых элементов контента.  
Шаги:
- Вычисляется степень сходства текущего пользователя с каждым синтетическим: учитывается разница в возрасте и количество общих жанров. Формула: `сходство = (общие_жанры / максимум_жанров) - (разница_возраста/10)*0.1`.
- Выбираются два синтетических пользователя с наибольшим сходством (соседи).
- Анализируются все оценки этих соседей: если оценка >= 4 и элемент подходит по возрасту и типу контента, то название элемента добавляется в общий список с накоплением суммы оценок.
- Список сортируется по убыванию суммы, выдаются первые 5 названий.

### Алгоритм 3 – по популярности
- Из базы выбираются все элементы заданного типа, подходящие по возрасту (без учёта жанров и целевой группы).
- Полученный список сортируется по полю `popularity` (число от 1 до 10) по убыванию.
- Первые 5 названий выдаются как рекомендации.

## Планы по доработке (будущие версии)

- Подключение внешней базы данных.  
  В текущей версии все данные хранятся в оперативной памяти (вектор структур). В следующих версиях планируется реализовать один из двух вариантов сохранения данных:
  - PostgreSQL – полноценная реляционная СУБД. Будут созданы таблицы `users`, `content`, `ratings`, `genres`. Это обеспечит возможность работы с большим объёмом информации.
  - Текстовый файл (CSV/JSON) – упрощённый вариант. Данные будут загружаться из файлов при запуске и сохраняться обратно. Это удобно для распространения без установки сервера.

- Замена синтетических оценок реальными пользовательскими рейтингами.  
  В текущем прототипе для коллаборативной фильтрации используются заранее заложенные оценки шести вымышленных пользователей. При внедрении базы данных каждый реальный пользователь сможет ставить оценки контенту, что повысит качество рекомендаций.

- Гибридный алгоритм рекомендаций.  
  Комбинация правил, коллаборативной фильтрации и популярности с возможностью динамической настройки весов.

- Графический интерфейс или веб‑интерфейс.  
  Улучшит удобство использования и наглядность рекомендаций.
